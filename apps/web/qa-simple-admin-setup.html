🧪 QA: Firebase Authentication Fix

❌ Issue Identified

Firebase Authentication Error: auth/configuration-not-found

Root Cause: Firebase project authentication is not enabled in Firebase Console

⚠️ Firebase Emulators Require Login

The Firebase emulators require firebase login which needs browser authentication.

Let's use a simpler approach for immediate testing.

🎯 Alternative Solution: Enable Firebase Console Authentication

Step 1: Enable Authentication

  1. Go to Firebase Console
  2. Click on "Authentication" in the left sidebar
  3. Click "Get started"
  4. Go to "Sign-in method" tab
  5. Click on "Email/password"
  6. Toggle "Enable" and click "Save"

Step 2: Create Admin User

  1. Go to "Users" tab in Authentication
  2. Click "Add user"
  3. Email: admin@macabre.com
  4. Password: admin123456
  5. Click "Add user"

Step 3: Set Admin Role in Firestore

  1. Go to "Firestore Database" in the left sidebar
  2. Click "Create database" (if not created)
  3. Choose "Start in test mode"
  4. Create a new collection called users
  5. Add a document with the user ID from step 2
  6. Add fields:
    • name: "Admin User"
    • email: "admin@macabre.com"
    • role: "ADMIN"
    • createdAt: timestamp (now)
    • updatedAt: timestamp (now)

✅ After Setup

Once you've completed the Firebase Console setup:

  1. Go to http://localhost:3000/login
  2. Login with: admin@macabre.com / admin123456
  3. Navigate to http://localhost:3000/spaces
  4. Test the "Post" button to submit a work
