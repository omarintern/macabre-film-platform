# Story 2.2: Content Submission Form

## Status
Done

## Story
**As a** Creator, **I want** a simple form to submit a new short text work, **so that** I can add it to my portfolio.

## Acceptance Criteria
1. A "Submit New Work" link is visible to authenticated 'CREATOR' users.
2. The form includes fields for a Title, the Body of the work, and Tags/Hashtags.
3. A required field exists to classify the submission as "Synopsis", "Scene Description", or "Other".
4. The Body text field enforces a 1000-character maximum.
5. The new work is saved and associated with the logged-in Creator.

**New Work Model Required** - No specific schema found in architecture docs, design based on acceptance criteria:
```typescript
model Work {
  id            String   @id @default(cuid())
  title         String
  body          String   // Max 1000 characters (enforced in validation)
  classification String  // "Synopsis", "Scene Description", or "Other"
  tags          String[] // Array of tags/hashtags
  creatorId     String
  creator       User     @relation(fields: [creatorId], references: [id])
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  
  @@map("works")
}
```

### API Specifications
**POST /api/works** endpoint requirements:
- Authentication: Require JWT token and CREATOR role
- Request body: `{ title: string, body: string, classification: string, tags: string[] }`
- Validation: title required, body max 1000 chars, classification must be valid enum
- Response: Created work object or error details
- Status codes: 201 (created), 400 (validation error), 401 (unauthorized), 403 (forbidden)
