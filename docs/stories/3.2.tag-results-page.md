## Story
**As a** user, **I want** to see a gallery of all works associated with a tag after clicking it, **so that** I can explore relevant content.

1. Clicking a tag/hashtag on the "Index" page navigates to a results page.
2. The page displays a gallery of all works associated with that tag/hashtag.
3. The gallery format is consistent with the "Spaces" page.

```typescript
interface Work {
  id: string;
  title: string;
  body: string;
  classification: string;
  tags: string[];  // Array of tag strings
  creatorId: string;
  createdAt: string;
  updatedAt: string;
  creator: {
    id: string;
    name: string | null;
    email: string;
  };
```

**Tag Interface** [Source: apps/web/lib/services/workService.ts]:
```typescript
interface Tag {
  name: string;
  count: number;  // Number of works using this tag
}
```

**Paginated Works Response** [Source: apps/web/lib/services/workService.ts]:
```typescript
interface PaginatedWorksResponse {
  success: boolean;
  works: Work[];
  pagination: {
    page: number;
    limit: number;
    total: number;
    totalPages: number;
    hasNext: boolean;
    hasPrev: boolean;
  };
}
```

### API Specifications
**Required New GET `/api/works/by-tag/[tagName]` endpoint** [Source: architecture/project-structure.md]:
- Location: `app/api/works/by-tag/[tagName]/route.ts`
- Purpose: Retrieve all works that contain the specified tag with pagination
- Response format: Same as existing `/api/works` endpoint (PaginatedWorksResponse)
- Must be publicly accessible (no authentication required for viewing)
- Should handle URL-encoded tag names properly
- Should return works ordered by creation date (newest first)

**API Response Format**:
```typescript
interface TagWorksResponse {
  success: boolean;
  works: Work[];
  pagination: {
    page: number;
    limit: number;
    total: number;
    totalPages: number;
    hasNext: boolean;
    hasPrev: boolean;
  };
  tagName: string;  // The tag being filtered by
}
```
