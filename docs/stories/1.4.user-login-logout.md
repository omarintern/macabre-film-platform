# Story 1.4: User Login & Logout

## Status
Done

## Story
**As a** registered user, **I want** to be able to log in and log out, **so that** I can securely manage my session.

## Acceptance Criteria
1. A login page with email and password fields is available.
2. Upon successful authentication, a secure session is created.
3. The user can access areas of the site that require authentication.
4. A logout mechanism is available that securely terminates the session.

### Authentication Flow Requirements
**Login Flow**:
1. User submits email/password on login page
2. Client-side validation and API call to /api/auth/login
3. Server validates credentials with bcrypt.compare()
4. Generate secure JWT token on successful authentication
5. Store token securely (httpOnly cookie)
6. Update client-side user session store
7. Redirect to protected area or dashboard

**Logout Flow**:
1. User clicks logout button/link
2. API call to /api/auth/logout with current token
3. Server invalidates token (blacklist or short expiry)
4. Clear client-side user session store
5. Redirect to login page or homepage

**Route Protection**:
- **Middleware**: `middleware.ts` to protect routes requiring authentication
- **Protected Routes**: Any routes outside of public pages and auth pages
- **Redirect Logic**: Unauthenticated users redirected to login page
