# Story 2.3: Display Works on Creator Profile

## Status
Done

## Story
**As a** user, **I want** to see a list of works on a Creator's public profile page, **so that** I can review their body of work.

## Acceptance Criteria
1. All short text works by a Creator are displayed on their public profile page.
2. Each work in the list displays its Title and its classification ("Synopsis", etc.).
3. The list is ordered with the most recently submitted work first.

### Data Models
**Work Interface** [Source: apps/web/lib/services/workService.ts]:
```typescript
interface Work {
  id: string;
  title: string;
  body: string;
  classification: string;
  tags: string[];
  creatorId: string;
  createdAt: string;
  updatedAt: string;
  creator: {
    id: string;
    name: string | null;
    email: string;
  };
}
```

**Database Schema** [Source: Story 2.2 implementation]:
- Work model exists in Prisma schema with proper Creator relationship
- `createdAt` field available for ordering (DESC for most recent first)
- Classification field contains "Synopsis", "Scene Description", or "Other"

### API Specifications
**Existing GET `/api/works` endpoint** [Source: apps/web/app/api/works/route.ts from Story 2.2]:
- Currently returns works by authenticated creator (own works)
- Uses JWT authentication with CREATOR role verification
- Returns works with creator relationship included

**Required Enhancement**:
- Either extend existing endpoint with query parameter `?creatorId={id}` for public access
- Or create new public endpoint `/api/works/by-creator/{userId}` 
- Must be publicly accessible (no authentication required for viewing)
- Should order results by `createdAt DESC`
