# Story 2.4: "Spaces" Gallery Page

## Status
Ready for Review

## Story
**As a** user, **I want** to visit a "Spaces" page, **so that** I can browse all the short text works submitted on the platform.

## Acceptance Criteria
1. A publicly accessible "Spaces" page is linked in the main navigation.
2. The page displays a gallery of all submitted short text works.
3. The gallery is paginated for performance.
4. Each work in the gallery displays its Title, classification, and the Creator's name.

### Data Models
**Work Interface** [Source: apps/web/lib/services/workService.ts]:
```typescript
interface Work {
  id: string;
  title: string;
  body: string;
  classification: string;
  tags: string[];
  creatorId: string;
  createdAt: string;
  updatedAt: string;
  creator: {
    id: string;
    name: string | null;
    email: string;
  };
}
```

**Pagination Response Interface** (to be created):
```typescript
interface PaginatedWorksResponse {

    limit: number;
    total: number;
    totalPages: number;
    hasNext: boolean;
    hasPrev: boolean;
  };
}
```

### API Specifications
**Required Enhancement to GET `/api/works` endpoint** [Source: apps/web/app/api/works/route.ts]:
- Currently requires CREATOR authentication and returns only user's own works
- Need to extend to support public access for all works
- Add query parameters: `?page=1&limit=20` for pagination
- Return works ordered by `createdAt DESC` (most recent first)
- Include creator information in response
- Must be publicly accessible (no authentication required for viewing)

**Pagination Query Parameters**:
- `page`: Current page number (default: 1)
- `limit`: Items per page (default: 20, max: 50)
