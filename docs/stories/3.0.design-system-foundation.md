## Story
**As a** development team, **I want** a comprehensive design system and technical foundation aligned with Parts 3 & 4 of the PRD, **so that** all future features can be built with consistent UX patterns, accessibility standards, and technical architecture.

## Acceptance Criteria
1. A centralized design system is established with consistent design tokens (colors, typography, spacing, components).
2. All UI components follow WCAG 2.1 AA accessibility standards.
3. The design aesthetic reflects the minimalist, content-forward approach inspired by Barnsworthburning.net.
4. Mobile-first responsive design patterns are implemented across all components.
5. Technical architecture is optimized for serverless deployment and performance.
6. Comprehensive testing strategy includes unit, integration, and accessibility testing.
7. All existing components are refactored to use the new design system.

### Refactoring Performed

- **File**: `apps/web/jest.config.js`
  - **Change**: Added `moduleNameMapper` configuration to both jsdom and node projects to resolve `@/lib/utils` import issues
  - **Why**: Jest was unable to resolve the `@/` path alias in the design system components
  - **How**: Ensures proper module resolution for all test environments

- **File**: `apps/web/components/ui/design-system/Button.test.tsx`
  - **Change**: Fixed keyboard navigation test to properly test focus management instead of simulating Tab key events
  - **Why**: The original test was trying to simulate keyboard events that don't actually move focus in the DOM
  - **How**: Tests actual focus management capabilities of the ButtonGroup component

- **File**: `apps/web/components/ui/design-system/Card.tsx`
  - **Change**: Converted empty interfaces to type aliases for all Card sub-components
  - **Why**: ESLint was flagging empty interfaces as equivalent to their supertypes
  - **How**: Uses type aliases which are more appropriate for simple type extensions
